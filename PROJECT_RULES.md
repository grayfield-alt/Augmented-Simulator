# 프로젝트 개발 지침서 (PROJECT RULES)

이 문서는 본 프로젝트 진행 시 AI 코딩 어시스턴트(Antigravity 등) 및 기여자가 항상 최우선으로 준수해야 하는 공통 가이드라인입니다.

## 1. 코드 스타일 (Code Style)
- **모듈 시스템**: 반드시 ES 모듈(`import`/`export`)을 사용하며, CommonJS(`require`)는 금지합니다.
- **문법**: 가능하면 구조 분해 할당(Destructuring assignment)을 적극적으로 사용합니다.
- **타입스크립트**: TypeScript 사용 시 반드시 `strict` 모드를 활성화하고 엄격한 타입 검사를 준수합니다.

## 2. 워크플로우 (Workflow)
- **타입 검사**: 코드 변경 후에는 반드시 `npm run typecheck` 명령어를 실행하여 타입 오류가 없는지 확인합니다.
- **테스트 실행**: 전체 테스트 스위트를 실행하지 말고, 관련된 단일 테스트만 실행합니다. (예: `npm test -- path/to/test`)
- **TDD 기반**: **[중요]** 항상 기능 구현 전에 테스트(Test)를 먼저 작성해야 합니다.

## 3. Git 컨벤션 (Git Conventions)
- **브랜치 네이밍**: 브랜치 생성 시 `feature/[작업명]` 또는 `fix/[이슈번호]` 형식을 따릅니다.
- **커밋 메시지**: 컨벤셔널 커밋(Conventional Commits) 규약을 준수합니다. (예: `feat:`, `fix:`, `docs:`, `refactor:`)
- **강제 푸시**: `main` 브랜치에 어떠한 경우에도 강제 푸시(Force Push, `-f`)를 금지합니다.

## 4. 백엔드 및 기타 주의사항
- **인증(Auth)**: `auth` 미들웨어는 `passport`가 `req.user`를 정상적으로 설정해야만 동작합니다.
- **데이터베이스**: DB 마이그레이션 작업은 반드시 `npm run migrate` 명령어를 통해서만 실행합니다.

---

## 5. 모바일 웹 배포 및 UI/UX 테스트 가이드 (커스텀 가이드)
본 프로젝트는 **웹 배포 환경 기반 모바일 플레이(스마트폰 세로 모드, Portrait Mode)** 를 전제로 개발됩니다.
따라서, UI/UX 개선 및 기능 개발 시 다음 사항을 항상 확인하고 테스트해야 합니다:

1. **세로형 레이아웃 필수 (Vertical Layout priority)**
   - 넓은 가로 화면 기준이 아니라, 좁은 세로 모바일 화면 기준에 맞춰야 합니다.
   - 카드(증강, 이벤트)나 컨테이너 요소가 가로로 나열(Side-by-side)되어 잘리지 않도록 `flex-direction: column` 등 세로 정렬을 강제합니다.
2. **오버레이 및 Z-index 정합성 (Overlay & Z-index Priority)**
   - 상태바, 모달, 가이드라인 등 화면을 덮는 오버레이(Overlay)들은 아래 요소(예: 로비 버튼)가 뚫고 나오지 못하도록 DOM 트리의 최상단 레벨에 배치(`body` 직속 권장)하고, 필요 시 `z-index: 99999 !important` 수준의 가장 높은 우선순위를 적용해야 합니다.
3. **버튼 정렬 중심부 배치 (Center Alignment for Core Actions)**
   - 모바일 환경에서의 조작 편의성을 위해 전투 UI의 '공격', '패링/방어', '회피' 등의 주요 액션 버튼은 화면의 하단 중앙 쪽에 정렬되도록 조정합니다. (`margin: 0 auto`, `left: 50%; transform: translateX(-50%)` 등의 중앙 정렬 기법 적용)
4. **터치 및 동적 피드백 확인**
   - 웹 테스트 시 브라우저에서 제공하는 [모바일 디바이스 시뮬레이션] 창에서 실제 터치 및 렌더링 환경과 동일하게 동작하는지 늘 검증해야 합니다.
